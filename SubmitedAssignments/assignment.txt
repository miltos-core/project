Περίληψη Εργασίας

Στο πλαίσιο της παρούσας εργασίας, καλείστε να αναπτύξετε το δεύτερο μέρος της διαδικτυακής εφαρμογής συστήματος πανεπιστημίου. Το δεύτερο μέρος της εργασίας εστιάζει στη δημιουργία του Dashboard των χρηστών (φοιτητών και καθηγητών).
Η εργασία σας θα πρέπει να περιλαμβάνει  την χρήση τεχνολογιών HTML/HTML5, CSS/CSS3 και JavaScript για την υλοποίηση ενός responsive UI, PHP για την υλοποίηση του Back-End τμήματος της εργασίας καθώς και export της βάσης δεδομένων (MySQL ή MariaDB) ώστε να μπορεί η εφαρμογή να τρέχει αυτοτελώς σε server (μέσω phpMyAdmin).


Τα παραδοτέα του δεύτερου μέρους της εφαρμογής θα είναι ο κώδικας που έχετε υλοποιήσει, το export της βάσης δεδομένων σας καθώς και μία αναφορά για το σύνολο της εργασίας (Μέρος 1 & 2).
Τα παραπάνω θα πρέπει να γίνουν .zip αρχείο και να υποβληθούν στο Moodle.
•	Η έκθεση Εργασίας - Project Report (περίπου 500 λέξεις, εξαιρούνται αναφορές και εικόνες)
•	Ένας σύνδεσμος (link) προς τα παραδοτέα της εργασίας σε Version Control System θα πρέπει να περιλαμβάνεται στο Project Report.
•	Η εργασία (Project) σε κώδικα περιλαμβάνει PHP, HTML/HTML5, CSS/CSS3, JS κώδικα.
•	Export της βάσης δεδομένων (MySQL ή MariaDB) ώστε να μπορεί η εφαρμογή να τρέχει αυτοτελώς χωρίς πρόβλημα.


Βασικές απαιτήσεις
1.	Συνεκτικότητα με το Πρώτο Μέρος: Η εφαρμογή θα πρέπει να είναι η συνέχεια της εφαρμογής που δημιουργήθηκε στο πρώτο μέρος.

2.	Συγχρονισμός Δεδομένων: Η εφαρμογή θα πρέπει να έχει τη δυνατότητα να συγχρονίζει δεδομένα με την κεντρική βάση δεδομένων, επιτρέποντας τον συνεχή συγχρονισμό των δεδομένων ανάμεσα σε καθηγητές και φοιτητές.

3.	Χρήση Βιβλιοθηκών ή Frameworks: Η εφαρμογή θα πρέπει να ενσωματώνει υπάρχοντα frameworks ή βιβλιοθήκες για να επιταχύνει και να διευκολύνει την ανάπτυξη, αλλά και να βελτιώνει την απόδοση της εφαρμογής.

4.	Σχεδιασμός Δοκιμών (Testing): Η εφαρμογή θα πρέπει να συνοδεύεται από διαδικασίες και σχέδια δοκιμών για την επαλήθευση της σωστής λειτουργίας της.

5.	Σταθερότητα και Απόδοση: Είναι σημαντικό η εφαρμογή να λειτουργεί αποτελεσματικά, προσφέροντας μια ομαλή εμπειρία στον χρήστη, για όλες τις πιθανές συσκευές (mobile/tablet, etc).

Τεχνικές απαιτήσεις

Στο δεύτερο μέρος της εργασίας, θα υλοποιηθεί κοινό περιβάλλον (dashboard) της εφαρμογής, το οποίο θα προσαρμόζεται ανάλογα με τον ρόλο του χρήστη (καθηγητής/φοιτητής). Θα πρέπει να φαίνονται καθαρά οι διαφορετικές λειτουργίες που προσφέρονται στον Φοιτητή και στον Καθηγητή μέσα από το ίδιο UI, με χρήση Role-Based Access Control (RBAC).

1.	Dashboard Φοιτητή:
•	RBAC έλεγχος, πρόσβαση μόνο σε φοιτητικές λειτουργίες.
•	Δυνατότητες:
o	Προβολή Μαθημάτων
o	Προβολή εργασιών
o	Υποβολή εργασιών
o	Προβολή βαθμολογιών

2.	Dashboard Καθηγητή:
•	RBAC έλεγχος, πρόσβαση μόνο σε λειτουργίες καθηγητών.
•	Δυνατότητες:
o	Δημιουργία και Διαχείριση Μαθημάτων
o	Ανάρτηση εργασιών
o	Προβολή υποβολών φοιτητών
o	Βαθμολόγηση φοιτητών ανά μάθημα (σύμφωνα με τις υποβολές εργασιών που θα βλέπει στο εκάστοτε μάθημα).

3.	Κοινό dashboard
•	Ο φοιτητής και ο καθηγητής μπαίνουν στο ίδιο dashboard.
•	Ανάλογα με τον ρόλο τους, βλέπουν διαφορετικό μενού και λειτουργίες.
•	Απόπειρα πρόσβασης σε σελίδες άλλου ρόλου, θα πρέπει να εμφανίζεται μήνυμα σφάλματος “Forbidden Action”.
•	Συγχρονισμός δεδομένων ανάμεσα και φοιτητές και καθηγητές (π.χ. μετά την υποβολή βαθμολογίας σε κάποιο μάθημα από κάποιο καθηγητή, θα πρέπει η πληροφορία αυτή να είναι διαθέσιμη στον εκάστοτε φοιτητή).

4.	Έκθεση Εργασίας
Η έκθεση εργασίας θα αφορά το σύνολο της εφαρμογής (Μέρος 1 & 2)

•	Κώδικας της Εφαρμογής: Ανάλυση του κώδικα και της βάσης δεδομένων με σκοπό να παρουσιαστούν τα κύρια μέρη της εφαρμογής.
•	Περιγραφή Διεπαφής: Screenshots που δείχνουν την υλοποίηση των διαφόρων λειτουργιών και την εμφάνιση της εφαρμογής σε διάφορες συσκευές.
•	Ενσωμάτωση Frameworks/Βιβλιοθηκών: Λίστα και σύντομη περιγραφή των frameworks ή βιβλιοθηκών που έχουν ενσωματωθεί στην εφαρμογή.
•	Τεκμηρίωση & Σχόλια Κώδικα: Συγκεκριμένα σχόλια που περιγράφουν τον σκοπό του κάθε τμήματος κώδικα, καθώς και οποιαδήποτε τεκμηρίωση που μπορεί να βοηθήσει τον εξεταστή να κατανοήσει τη λειτουργία της εφαρμογής.
•	Παρουσίαση των δοκιμών που εκτελέστηκαν και τεκμηριώνουν την λειτουργικότητα της εφαρμογής.
•	Οποιαδήποτε άλλη σημαντική πληροφορία.

5.	Προφορική Εξέταση
•	Διάρκεια 15 λεπτά
•	Περιλαμβάνει επίδειξη λειτουργιών του dashboard είτε live είτε με ένα demo video.
•	Εξήγηση των βασικών λειτουργιών και της βάσης δεδομένων.
•	Απάντηση στις ερωτήσεις του καθηγητή που θα αφορούν στη λειτουργικότητα και τη θεωρία του μαθήματος.

Περαιτέρω οδηγίες για την εργασία
Οι παρακάτω οδηγίες θα σας βοηθήσουν να πάρετε ένα καλό βαθμό στην εργασία σας:
•	Η έκθεση της εργασίας σας δεν πρέπει να περιλαμβάνει γλωσσάριο, abstract ή περίληψη. Πρέπει να εστιάσετε ώστε να καλυφθούν τα (προ-απαιτούμενα)
στοιχεία της εργασίας σας που πρόκειται να βαθμολογηθούν.
•	Παραρτήματα και αναφορές δεν συμπεριλαμβάνονται στο σύνολο των λέξεων.
•	Διαβάστε προσεκτικά την περιγραφή της εργασίας και την ανάλυση της βαθμολογίας. Δεν θα βαθμολογηθείτε για κώδικα που δεν έχετε γράψει εσείς.
 

Τι πρέπει να υποβληθεί
Προς το παρόν γίνεται αποδεκτό μόνο Microsoft Word σε DOC/DOCX ή RTF format για την έκθεση. Σχεδόν όλοι οι κειμενογράφοι παράγουν το συγκεκριμένο format.

Η υποβολή σας θα πρέπει να είναι οργανωμένη στα παρακάτω αρχεία και φακέλους:
•	Ένας φάκελος με την ονομασία prototype, που θα περιέχει όλα τα αρχεία που θα χρησιμοποιηθούν στην εφαρμογή. Πρέπει να συμπεριληφθούν οι
υποφάκελοι με τη ορθή δομή της εφαρμογής (για παράδειγμα ένας φάκελος
pictures που θα περιέχει τις εικόνες)

•	Ένας φάκελος με την ονομασία notes (προαιρετικός) που θα περιλαμβάνει οποιαδήποτε επιπρόσθετη πληροφορία πιστεύετε ότι χρειάζεται για τον εξεταστή.

•	Ένα αρχείο που θα φέρει τον αριθμό μητρώου σας που θα περιέχει την έκθεση εργασίας.
Τα παραπάνω θα πρέπει να συμπιεστούν (zip) πριν την υποβολή και το παραγόμενο αρχείο να ονομαστεί σύμφωνα με τις οδηγίες που σας έχουν δοθεί σχετικά με την κατάθεση (π.χ. ind_ass_[τον αριθμό μητρώου σας]).

 
Παρατηρήσεις και για το Μέρος Α’ και για το Μέρος Β’

•	Μην συμπεριλάβετε επιπλέον zip αρχεία μέσα στο τελικό zip, εκτός και αν κρίνετε ότι είναι απολύτως απαραίτητο.
•	Το τελικό αρχείο δεν πρέπει να ξεπερνάει τα 20MB. Αν συμβαίνει κάτι τέτοιο πιθανό να οφείλεται στο μέγεθος των εικόνων τις οποίες θα πρέπει να
αναπροσαρμόσετε (μικρύνετε) με χρήση λογισμικού εικόνας.

•	Βεβαιωθείτε ότι υποβάλετε τα σωστά αρχεία, ότι είναι ολοκληρωμένα και ότι δεν βγάζουν μήνυμα «κατεστραμμένο» (corrupt). Ελέγξτε την υποβολή σας «κατεβάζοντας» το αρχείο που υποβάλατε σε ένα ξεχωριστό κατάλογο και βεβαιωθείτε ότι λειτουργεί.
 
ΠΡΟΣΟΧΗ
•	Δεν υπόκεινται στα καθήκοντα των εξεταστών να διορθώνουν μη λειτουργικούς («κατεστραμμένους») φακέλους και αρχεία. Σε αυτή την περίπτωση θα λάβετε μηδενική βαθμολογία.
•	Οι εξεταστές δεν θα διορθώσουν κανένα λάθος που θα σχετίζεται με την
πρόσβαση στην ιστοσελίδα σας. Αν συμβεί κάτι τέτοιο θα χάσετε βαθμούς.
•	Οι εξεταστές θα βαθμολογήσουν το κείμενο χρησιμοποιώντας είτε MicrosoftOffice ή LibreOffice. Καμία προσπάθεια δεν θα γίνει να βαθμολογηθούν κείμενα σε διαφορετικά format πέρα από αυτά που αναφέρθηκαν.
•	Μην υποβάλλετε οτιδήποτε δεν αποτελεί μέρος της εργασίας σας.

Αν αποτύχετε να ακολουθήσετε τις παραπάνω οδηγίες, αυτό πιθανώς θα οδηγήσει σε απώλεια βαθμών.
